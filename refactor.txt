## 10. REMOVE FROM CURRENT UI

* Gradients
* Glow effects
* Oversaturated alert backgrounds
* Icon background tiles
* Redundant labels

---

This spec is **complete and sufficient** to reproduce a clean, professional, enterprise-grade dashboard without ambiguity.

UI critique and concrete improvements:

**Hierarchy**

* Reduce header height by ~20%. Current header steals vertical space.
* Increase contrast between section titles and background. “Forecast Trends” is visually weak.
* Promote alerts to primary severity ordering: Critical → High → Medium. Current visual weight is inconsistent.

**Cards (Top Metrics)**

* Align numeric emphasis: numbers first, labels second. Increase number font size by 20–30%.
* Remove pastel icon backgrounds or unify to a single semantic color system.
* Add trend indicators (↑ ↓ %) inside each card; static numbers lack context.

**Color System**

* Alerts panel overuses saturated gradients. Use flat colors with a single accent stripe.
* Define strict semantic colors:

  * Red = risk/action required
  * Amber = warning
  * Blue = informational
* Reduce glow/shadow effects; they reduce readability.

**Chart Area**

* Y-axis labels are too faint. Increase opacity.
* Confidence band opacity too high; reduce to ~20–25%.
* Add a clear legend or inline annotation for “forecast vs actual.”
* Enable hover crosshair and value pinning.

**Alerts Panel**

* Too wide. Compress width by ~15% to prioritize analytics.
* Standardize alert cards:

  * Title
  * One-line cause
  * One-line impact
* Remove decorative badges unless they convey state changes.

**Spacing & Alignment**

* Normalize padding across all cards. Current vertical rhythm is uneven.
* Increase whitespace between sections, reduce inside cards.

**Navigation**

* “New Forecast” should be primary action; “Dashboard” should not be highlighted while active.
* Group “Alerts” and “History” under a utility menu to reduce top-bar noise.

**Typography**

* Use one font weight for body, one for headings. Current mix feels arbitrary.
* Increase line-height in alerts text for scanability.

**Data Density**

* Add compact table view toggle under chart for power users.
* Enable timeframe switch (4w / 8w / 12w) inline, not hidden.

**Overall**

* Simplify visuals.
* Increase signal-to-noise.
* Make risk and action states unmistakable.
